DBAddFilter
SINTAXIS
[DBAddFilter] Condition [ | FielsNoInfo [ | _PDFCONDITIONHIDDEN  [ | _PDFCONDITIONHIDDEN ... ] ] ]
DESCRIPCION
Condición que se añade a la cláusula "where", en el caso de que sea un grupo de fichas la etiqueta se pondrá en el GDF (script principal del grupo de fichas).
Se pueden poner variables encerradas entre llaves, por ejemplo: "{$_Node}" si se tiene que utilizar comillas se deben de utilizar las comillas simples.
También puede ser una función, sin parámetros, que devolverá la condición a añadir.
hay una forma especial para buscar registros con condiciones de otras tablas, poniendo un "=" antes del "where" a añadir y poniendo el alias del campo de la tabla padre como "#." o devolverlo desde una función.
[DBAddFilter] =exists (select [campo] from [tabla] where #.[campo]=10000)
[DBAddFilter] =FuncUser()
Si se quiere utilizar la variable $_DBADDFILTER" para cambiar el filtro de forma dinámica es posible tener conflicto con los alias de los campos en este caso todos los campos que necesiten alias de la tabla principal de definirán con el alias "#" siendo por ejemplo $_DBADDFILTER = "#.campo in ( select ...)"
PARAMETROS
Condition  Condición SQL. Si hay select o se utilizan mas de una tabla y el nombre de los campos a usar está en varias tablas se le tendrá que anteponer el alias siendo "A" el alias de la tabla principal y cada select que se utilice de otra tabla "B", "C", etc.
Se puede utilizar esta etiqueta para los subselect cuando estos los ejecutan un "ldf".  
FielsNoInfo  Lista de campos a no informar en los listados ni PDF cuando se muestran las condiciones del listado.
EJEMPLO
[DBAddFilter] estado=3
[DBAddFilter] gs_node={$_Node}
[DBAddFilter] HesdeHace2Dias()
[DBAddFilter] cd_prov in (select cd_auto from auto)
[DBAddFilter] =exists (select cd_auto from auto where cd_auto=#.cd_prov)
