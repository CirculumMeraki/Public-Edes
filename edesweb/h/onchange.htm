OnChange
SINTAXIS
[OnChange] Mode | Field [,Field,..] | JavaScript [[ | FuncUsuario() | SeEjecutaEnLaCarga ]]
...
DESCRIPCION
Cada vez que cambie el valor del campo se ejecutara el código JavaScript asociado.
En el tercer parámetro podemos indicar código JavaScript. Se puede repetir la etiqueta con el mismo campo sumandose todos los script.
Hay una función muy utilizada en esta etiqueta que es "eCallSrv()" que si se quiere que no se ejecute en la carga de la página la llamaremos con barra baja delante de la función "_eCallSrv()".
PARAMETROS
Mode  
Field  Campo o lista de campos sobre el qué deseamos crear el evento.
JavaScript  Código JavaScript, si la función a ejecutar empieza por "_" no se ejecutará en la carga de la página
FuncUsuario()  Es una función de usuario que se llamará cuando se recarge un subselect, esta función puede llevar parámetros y variables php entre llaves.
SeEjecutaEnLaCarga  Por defecto es true, si ponemos false no se ejecutará en la carga, en este caso no haría falta que la función de javascript empiece por "_"
Parametro "JavaScript" : Puede ser o código javascript o una llamada a un fichero "SDF" que es un fichero de definición para la selección de datos de otras tablas, por ejemplo si lo pones en un campo de CIF de empresa al rellenar el CIF irá a este fichero a recuperar por ejemplo en nombre de la empresa.
La forma de definirlo sería la siguiente: [OnChange] a,mR | cif | "cif.sdf?CIF="+this.value
donde "cif.sdf" sería el fichero de definición que busca el nombre de la empresa y se le pasaría el parámetro "CIF=this.value".
Y el fichero de definición "cif.sdf":

select nm_empresa from empresa where cif='{$CIF}'
Count = 1
Put: nm_empresa
Count = 0
Clear: cif, nm_empresa
Menssage: La empresa no existe

Comandos de los ficheros SDF:
- Variables pasadas por url se utilizan de la siguiente forma:  '{$...NombreVariable...}'
- Variables sin nombre tomado los valores de la linea de argumentos:  '{$argv[0]}'
- Todos las sentencias "select" se guarda su resultado en la matriz asociativa "$_vF", utilizandola de la siguiente forma:  '{$_vF['cd_prov']}'

FuncionDeUsuario().- En la primera linea que devolverá el nº de registros encontrados o un nº negativo, este valor se utiliza para saber que lineas de código se ha de ejecutar, lineas "Count ...".

select * from postal where cd_postal='{$CPOSTAL}'

empty:
 clear: cd_postal, nm_postalCount = 0

Count = 0
 clear: cd_postal, nm_postal
 help: Código postal DESCONOCIDO
 focus: cd_postal

Count = 1
 put: nm_postal

Count < 50   // Elegir en un listado de entre los 50
 list: postal.edf&_FILTER=cd_postal='{$CPOSTAL}'&_ASSIGN=cd_postal=cd_postal,nm_postal=nm_postal
Count >= 50// Ampliar condiciones de busqueda desde una ficha
 tab: postal.edf&_FILTER=cd_postal='{$CPOSTAL}'&_ASSIGN=cd_postal=cd_postal,nm_postal=nm_postal
Este ejemplo es de un uso normal, la secuencia es:
- Busca un registro y en función del número de registros encontrados podrá hacer:
- Si ha quitado el valor del campo:
Vaciará los campos
- Si no encuantra ningún registro:
Vaciará los campos, mostrará un mensaje y se colocará en el campo que no ha encontrado el valor.
- Si encuentra sólo uno:
Rellenera el resto de campos.
- Si encuentra menos de 50 y mas de 1
Mostrarña un listado con los encontrados para seleccionar uno.
- Y si encuentra 50 o mas:
Mostrará una ficha de búsqueda, con los datos de busqueda asignados por defecto, para que se metan mas condiciones y buscar para elegirlo.
