PDFTh
SINTAXIS
[PDFTh] Title1 [, Title2] ..... [, TitleN] [ | campo=etiqueta, ...]  [ | campo=NomFuncion() ; [ campo=NomFuncion()] ]
DESCRIPCION
Título de las cabeceras del listado en PDF, si se deja en blanco no pondrá nada, si se pone un "=" o no se ha definido la columna pondrá la etiqueta del formulario.  El segundo parámetro es para definir label de campos que no está en la definición del listado ni en la DB "gs_campo" y el tercer parámetro es para mostrar un valor distinto en el valor de la condición.
PARAMETROS
Title1: Es el TH del listado
Campo=Etiqueta: Cuando a un listado lo sacas con condiciones y los campos que condiciona no están definidos en "gs_campo" se pueden difinir con este parámetro.
EJEMPLO
[PDFTh] EMPRESA,R.SOCIAL,FECHA,IMPORTE
[PDFTh] =,R.SOCIAL

[PHPIni] *
function LitaralEstado( $v ){
if( $v=='[ABC]' ) return 'ACTIVOS';
if( $v=='[DE]' ) return 'EN ESPERA';
sqlQuery( "select nm_estado from estado where cd_estado='{$v}'" );
$row = sqlRow();
return $row[0];
}

[PDFTh] || estado=LitaralEstado();

En el segundo ejemplo solo se cambiará el nombre a la columna dos, las otras tres serán igual que la etiqueta del formulario.
En el tercer caso varios de los posibles valores de condicionar el campo "cd_estado" es por un grupo de registros que no tienen un literal específico para definirlo para lo cual se llama a la función.
