DESCRIPCION

   Consultas   




1. El servidor envía un formulario para que el usuario introduzca los filtros o condiciones. (estado c) El usuario lo rellena y lo envía al servidor.  
 

 

 
2. Cuando el servidor recibe los datos:  
 
a. Si el servidor puede localizar un registro único que se identifique con los datos que el usuario ha introducido entonces le devuelve la ficha para visualizarla. (estado cR)  
 

 

 
b. Si el servidor ve que hay varios registros que cumplen las condiciones, entonces le envía una lista para que escoja el registro que desea visualizar. (estado L). Cuando el usuario selecciona un registro de la lista, el servidor actúa igual que en el caso anterior, le envía la ficha para visualizarla. (estado cR)  
 

 

 

 
a. Si el servidor no encuentra ningún registro con las condiciones que el usuario ha enviado le muestra un mensaje advirtiéndoselo, en este caso termina el proceso.  
 

 

 
3. Cuando el usuario termina de visualizar los datos, regresa automáticamente al primer punto por si desea seguir consultando. Lógicamente en las consultas no se actualiza la base de datos.
