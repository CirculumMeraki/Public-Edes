FormAction
SINTAXIS
[FormAction] Mode | Action [ | AnAction ]
DESCRIPCION
Modifica el valor por defecto de la acción del formulario  
En el "Action" del formulario, se trata de la URL o función a ejecutar en vez de la que tiene por defecto.
Podemos poner una variable sin "$" encerrada entre llaves, es una variable php, esta variable puede estar dentro de una cadena.
Esta etiqueta por defecto desactiva la etiqueta [FormButtons].
Si el parámetro "Action" es una función desde esta no se podrá llamar a "eOkTab()", si hiciera falta en lugar de usar la etiqueta [FormAction] habría que usar la etiqueta [MsgSubmit].
Esta etiqueta es anulada por [OptionsInList].
PARAMETROS
Mode: Modo en la que se va a ejecutar la etiqueta.
Action: Orden a ejecutar, puede contener tres valores:
1.- Script a ejecutar
2.- Función a ejecutar
3.- Una variable JavaScript que contendrá el valor a ejecutar.
Si el parámetro "Action" termina en ":" se le añadirá automáticamente en nombre del script y si no tuviera "edes.php?" también.
4.- La palabra clave "SubList" será como pulsar con el botón derecho, dividiendote la pantalla en dos partes con la ficha de busqueda en la parte superior y el listado en la inferior, teniendo las opciones de mantenimiento de fichas en el listado (alta, baja, consulta y modificación).
5.- La palabra clave "NoSubList" desactiva la ejecución de la sublista con el botón derecho.
AnAction: Por defecto el botón de submit puede tener varias acciones, la palabra clave "AnAction" deja solo la acción principal.
EJEMPLO
[FormAction] c  | {_ExeExtract}
[FormAction] cR | edes.php?E:facura/recuento.php
[FormAction] a  | FunctionJS()
[FormAction] c  | edes.php?Fc:auto.edf
[FormAction] c  | edes.php?Ll:
[FormAction] c  | Ll:
[FormAction] c  | Fc:auto.edf
[FormAction] c  | SubList
[FormAction] c  | DOWNLOAD
[FormAction] *  |  | AnAction
