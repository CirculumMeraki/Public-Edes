   Modificaciones   




1. Al igual que en las bajas, el servidor le envía un formulario para tratar de localizar el registro deseado. (estado m) El usuario rellena las condiciones necesarias para localizarlo y lo acepta.  
 

 

 
2. Cuando el servidor lo recibe pueden pasar tres cosas:  
 
a. Si el servidor puede localizar un registro único que se identifique con los datos que ha introducido el usuario, le envía un formulario para la modificación de dicho registro. (estado mR)  
 

 

 
b. Si el servidor ve que hay varios registros que cumplen las condiciones, entonces le envía una lista para que escoja el registro que desea modificar. (estado L). Cuando el usuario selecciona un registro de la lista, el servidor actúa igual que en el caso anterior, le envía el formulario de modificación. (estado mR)  
 

 

 

 
c. Si el servidor no encuentra ningún registro con las condiciones que el usuario ha enviado le muestra un mensaje advirtiéndoselo, en este caso termina el proceso.  
 

 

 
2. Cuando el usuario acepta el formulario, el servidor procesa la actualización (estado M) y al finalizar se encarga de volver al primer punto por si el usuario desea seguir modificando más registros.
