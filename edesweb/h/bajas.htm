   Bajas
   




1. El servidor le envía un formulario para rellenar unos datos y tratar de localizar el registro que desea borrar. (estado b) El usuario lo rellena y lo acepta.  
 

 

 

2. Cuando el servidor lo recibe pueden pasar tres cosas:  
 
a. Si el servidor puede localizar un registro único que se identifique con los datos que ha introducido el usuario, le envía un formulario para que confirme la eliminación de dicho registro. (estado bR)  
 

 

 
b. Si el servidor ve que hay varios registros que cumplen las condiciones, le envía una lista para que escoja el registro que desea dar de baja. (estado L). Cuando el usuario selecciona un registro de la lista, el servidor actúa igual que en el caso anterior, le envía un formulario para que confirme la eliminación del registro seleccionado. (estado bR)  
 

 

 
c. Si el servidor no encuentra ningún registro con las condiciones que el usuario ha enviado le muestra un mensaje advirtiéndoselo, en este caso termina el proceso.  
 

 

 
3. Cuando el usuario confirma la eliminación del registro, el servidor procesa la actualización (estado B) y al finalizar se encarga de volver al primer punto por si el usuario desea seguir dando de baja más registros.
