OptionsInList
SINTAXIS
[OptionsInList] Opciones [ | Icon | userFunction | label ... ]  [ | Condition/Funcion ]
DESCRIPCION
Ejecutar un listado con opciones de mantenimiento, la opción se ejecutará como "edes.php?Lcl:..", el modo en que se ejecuta indica la opción por defecto a ejecutar en el listado, también puede ser "ml", o "bl".
En los script "gdf" se pondrá la etiqueta en el script de la primera solapa, que es de la que se genera el listado.
Si se quiere poder filtrar al entrar se ejcutará con "edes.php?Lgl:.." y activará la condición si existe.
En "Opciones" se puede combinar el "*" con cualquier opción así como desactivarla ponieno un signo menos delate de la opción "*-b" todas las opciones menos la de borrar.
La coma "," delimitará mediante una barra vertical las opciones.
La "Condición" puede ir en el 2 o 5 parámetro en función de si tiene la opción de usuario o no, la condición puede ser texto o el nombre de una función terminada con "()" que devuelva la cadena del where.
En "Opciones" si ponemos una "o" será una o varias opciones del usuario, cada opción constará de tres parámetros: el icono, la función de usuario y el label, estas opciones funcionan igual que las de gestión pulsas la opción y luego un registro de la tabla, si se quiere ejecutar la opción directamente se pondrá el caracter ">" delante del nombre de la función y esta enviará el objeto "this".
Esta etiqueta anula a [FormAction].
Esta etiqueta al ejecutar listados se activa una variable para saber el modo en que se ejecuta dicha etiqueta:
     $_OptionsInListMode:
          LR....: LastRecords
          NLR-NR: NoLastRecords-NoRecords
          NLR-L.: NoLastRecords-List
          NLR-AF: NoLastRecords-AddFilter
PARAMETROS
 
EJEMPLO
[OptionsInList] *
Establece las opciones de "abcmfou", "f" de Filter, "u" de "últimos registros gestionados" y la "o" representa una opción definida por el usuario.
EJEMPLO
[OptionsInList] mc
Solo se permiten las opciones de Modificar y Consultar
EJEMPLO
[OptionsInList] bcmafou | email | uFunction | Mandando email
[JSIni] *
function uFunction(oTR, pk1 [,...]){
     // oTR: objeto TR pulsado
     // pk1: primer valor de la etiqueta [DBIndex]
}
La "o" define una opción del usuario
